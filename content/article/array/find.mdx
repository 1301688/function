# はじめに、なぜそうなったのか？から学べばキャッチアップしやすい

## 結論(conclusion)

- find()を使用する

- find とは
  - 配列内から特定の要素を抜き出す時に使用する
  - 提供されたテスト関数を満たす配列内の最初の要素の値を返すメゾット
  - ID 検索の時に使用することが多い
  - 「条件に合う最初のひとつを探す」という前提条件がある

## 構文(sintax)

```js
array.find(callback(element[, index[, array]]) {
  // 実行する文
}[, thisArg]);
```

## 引数(argument)

- 1. callback = コールバック関数　配列内の各要素に対して実行する関数　次の 3 つの引数をとる
  - element = 配列内で現在処理されている要素
  - index = 配列内の currentValue の添字
  - array = forEach()が呼び出されている配列
- 2. thisArg = callback 内で this として使われるオブジェクト

## 返り値(return value)

- 配列の中で、提供されたテスト関数を満足する最初の要素の値
- 見つからなかった場合は undefined を返す

## ユースケース

- 「条件に合うひとつ」ではなく、「条件に合う全部」の場合は filter()

- 「条件に合うひとつの位置」を探すなら findIndex()

- 「条件に合うひとつ」があるかどうかだけわかれば良いなら some()

- 「条件に合う」ものを何度も何度も探すなら map()
