# はじめに、なぜそうなったのか？から学べばキャッチアップしやすい

## 結論

- join( )を使用する

- join とは
  - 配列の要素を順に連結してひとつづきの文字列にする為に使用する
  - もとの配列を変更せず、新しい文字列を返す非破壊的メゾット

## 構文(sintax)

```js
array.join([separator])
```

## 引数(argument)

- 1. separator = 配列の各要素を区切る文字列を指定します。
- separator は、必要であれば文字列に変換されます。省略した場合、配列の要素はカンマ (",") で区切られます。
- separator に空文字列を渡した場合、すべての要素の間が区切り文字無しで繋がれます。

## 返り値(return value)

- 配列の全要素が連結された文字列
- arr.length が 0 だった場合、空の文字列が返される

## ユースケース

- 配列  内の要素を多用な形でつなぐとき

- 配列風オブジェクトを連結するとき

- 「条件に合うひとつ」があるかどうかだけわかれば良いなら some()

- 「条件に合う」ものを何度も何度も探すなら map()
